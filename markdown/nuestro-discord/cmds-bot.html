warn - Warn a member
Usage:
warn <member> <reason>

Gives member a warning, usually for breaking the rules.

Permission:
Kick members or Moderator role
kick - Kick a member from the server
Usage:
kick <member> [reason]

Kick a member from the server.

Example:
kick @user
kick @user Did a bad thing
Permission:
Kick members or Moderator role
voice-kick - Disconnects a member from a voice channel
Usage:
voice-kick <member> [reason]

Disconnect a member from a voice channel.

Example:
voice-kick @user
voice-kick @user Did a bad thing
Permission:
Move members or Moderator role
mute - Mute a member from text channels
Usage:
mute <member> [duration] [reason]

Mutes a member from the server. This prevents them from sending messages.

If no duration is given they will be muted until un-muted.
Duration format examples: 1s, 1m, 1h, 1d, 1w, 1mth, 1y.

Example:
mute @user
mute @user 3d
mute @user 1w Did a bad thing
Permission:
Kick members or Moderator role
soft-ban - Kick a member from the server and delete their messages
Usage:
soft-ban <user> [reason]

Kicks a user from the server and deletes a days worth of their messages.
(Specifically, this bans and then immediately un-bans them)

Permission:
Kick members or Moderator role
temp-ban - Temporarily ban a user from the server
Usage:
temp-ban <user> <duration> [days delete] [reason]

Bans user temporarily for the duration specified and deletes last x days worth of messages.

If days is not a number, it's treated as the first word of the reason.
Minimum 0 days, maximum 7. Defaults to 0.

Example:
temp-ban @user 1d
temp-ban @user 1w Did a bad thing
temp-ban @user 1mth 1 Did a very bad thing
Permission:
Ban members or Moderator role
ban - Ban a member from the server
Usage:
ban <user> [days delete] [reason]

Bans user and deletes last x days worth of messages.

If days is not a number, it's treated as the first word of the reason.
Minimum 0 days, maximum 7. Defaults to 0.

Example:
ban @user
ban @user Did a bad thing
ban @user 1 Did a very bad thing
Permission:
Ban members or Moderator role
un-ban - Unban a user from the server
Usage:
un-ban <user> [reason]

Unbans a user from the server.

Permission:
Ban members or Moderator role
un-mute - Unmutes a muted member
Usage:
un-mute <member> [reason]

Unmute a muted member. This allows them to send messages again.

Permission:
Kick members or Moderator role
view-case - View information of a punishment
Usage:
view-case <case ID>

View details of a user's punishment. Don't include the # when specifying the case ID.

remove-case - Remove a punishment from a user
Usage:
remove-case <case ID> [reason]

Remove a case ID from a user. If there isn't another active mute or ban case the user will also be un-muted/un-banned.

Permission:
Kick members or Moderator role
punishments - Shows the history of a users punishments
Usage:
punishments <user> [page]

Shows information regarding a user's punishments in the server (warn / mute / kick / soft-ban / temp-ban / ban).

reason - Edit the reason of a punishment
Usage:
reason <case ID> <new reason>

Edit the reason for an existing punishment.

Example:
reason 25 Spam
Permission:
Kick members or Moderator role
purge - Delete messages
Usage:
purge <limit> [user | -bots | -users | -links | -invites | -embeds | -images | -files | -mentions | -pins | -silent]

Delete many messages quickly. Any pinned messages are not deleted unless the -pins filter is given.
The maximum limit is 1000, and multiple filters can be used.

If a filter is provided, messages within the limit will be checked if they match all filters.

Use the -silent filter to delete the purge message and confirmation.

Example:
purge 1000
purge 100 -images
purge 10 -bots -embeds
purge 50 -invites @user
Permission:
Manage messages or Moderator role
lock-down - Lockdown a channel or the whole server
Usage:
lock-down [channel | server] [duration]

If no duration is provided the channel/server will be locked down until the command is used again.

Channel Lockdown

The channel will be locked down for anyone that doesn't have the Send Messages permissions set to allow in the channel settings.

Server Lockdown

This denies the Send Messages permission for the first role that allows Send Messages.

It will lockdown all channels for all users/roles that don't allow the Send Messages permission in the role settings or channel settings.

Example:
lockdown #general 20s
lockdown #general
lockdown server
Permission:
Manage channels or Moderator role
slow-mode - Change the slowmode duration of a channel
Usage:
slow-mode [channel] <duration>

Set or change the slowmode duration (between 0s and 6h).
Members will be restricted to sending one message per this interval, unless they have Manage Channel or Manage Messages permissions.

Example:
slow-mode 0s
slow-mode #general 3m
Permission:
Manage channels or Moderator role
